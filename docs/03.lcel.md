# LangChain Expression Language (LCEL)

El Lenguaje de Expresión de LangChain (LCEL) es una abstracción de algunos conceptos interesantes de Python en un formato que permite una capa de código "minimalista" para construir cadenas de componentes de LangChain.

LCEL viene con soporte sólido para:

1. Desarrollo ultrarrápido de cadenas.
2. Características avanzadas como streaming, ejecución asíncrona, paralela y más.
3. Integración fácil con LangSmith y LangServe.

## Sintaxis de LCEL

Para entender la sintaxis de LCEL, exploremos una cadena simple usando la sintaxis tradicional de LangChain. Se inicializa una `LLMChain` simple usando **Claude 2.1**.

```python
from langchain.chat_models import ChatAnthropic
from langchain.prompts import ChatPromptTemplate
from langchain.schema.output_parser import StrOutputParser

ANTHROPIC_API_KEY = "<<ANTHROPIC_API_KEY>>"

prompt = ChatPromptTemplate.from_template(
    "Dame un pequeño informe sobre {topic}"
)
model = ChatAnthropic(
    model="claude-2.1",
    max_tokens_to_sample=512,
    anthropic_api_key=ANTHROPIC_API_KEY
)  # intercambia Anthropic por OpenAI con `ChatOpenAI` y `openai_api_key`
output_parser = StrOutputParser()
```

Usando esta cadena podemos generar un pequeño informe sobre un tema en particular, como "Inteligencia Artificial" llamando al método `chain.run` en una `LLMChain`:

```python
from langchain.chains import LLMChain

chain = LLMChain(
    prompt=prompt,
    llm=model,
    output_parser=output_parser
)

# y ejecutar
out = chain.run(topic="Inteligencia Artificial")
print(out)
```

Con LCEL se puede crear la cadena de manera diferente usando operadores de tubería (|) en lugar de `Chains`:

```python
lcel_chain = prompt | model | output_parser

# y ejecutar
out = lcel_chain.invoke({"topic": "Inteligencia Artificial"})
print(out)
```

La sintaxis aquí no es típica de Python pero utiliza únicamente Python nativo. El operador `|` simplemente toma la salida de la izquierda y la alimenta a la función de la derecha.

### **¿Cómo funciona el operador de tubería?**

Para entender qué sucede con LCEL y el operador de tubería, mire las siguientes funciones compatibles con tuberías.

Cuando el intérprete de Python ve el operador `|` entre dos objetos (como a | b), intenta alimentar el objeto a en el método __or__ del objeto b. Eso significa que estos patrones son equivalentes:

```python
# enfoque de objeto
chain = a.__or__(b)
chain("some input")

# enfoque de tubería
chain = a | b
chain("some input")
```

Para lograr esto, podemos construir una clase `Runnable` que consuma una función y la convierta en una función que se pueda encadenar con otras funciones utilizando el operador de tubería `|`. Aquí hay un ejemplo de cómo se podría implementar:

```python
class Runnable:
    def __init__(self, func):
        self.func = func

    def __or__(self, other):
        def chained_func(*args, **kwargs):
            # la otra función consume el resultado de esta función
            return other(self.func(*args, **kwargs))
        return Runnable(chained_func)

    def __call__(self, *args, **kwargs):
        return self.func(*args, **kwargs)
```

Implementemos esto para tomar el valor 3, sumar 5 (dando 8) y multiplicar por 2 — dándonos 16:

```python
def add_five(x):
    return x + 5

def multiply_by_two(x):
    return x * 2

# wrap the functions with Runnable
add_five = Runnable(add_five)
multiply_by_two = Runnable(multiply_by_two)

# run them using the object approach
chain = add_five.__or__(multiply_by_two)
chain(3)  # should return 16
```

Usando __or__ directamente obtenemos la respuesta correcta, intentemos usar el operador de tubería `|` para encadenarlos juntos:

```python
# chain the runnable functions together
chain = add_five | multiply_by_two

# invoke the chain
chain(3)  # we should return 16
```

Con cualquiera de los dos métodos, obtenemos la misma respuesta y, en esencia, esta es la lógica de tubería que utiliza LCEL al encadenar componentes. Sin embargo, esto no es todo lo que ofrece LCEL, hay más.
